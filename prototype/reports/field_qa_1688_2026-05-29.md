## Fixed During QA

1. Image format normalization.

   Symptom: `image.png` returned `NO_VISUAL`; WB photo endpoint returned 500 model inference errors. The same image converted to JPEG returned 256 visual matches.

   Fix: `app/services/wb_visual_search.py` now converts PNG/WebP/HEIC/etc. to plain JPEG before upload.

2. False benchmark for doll strollers.

   Symptom: `Игрушки/Коляски для кукол` matched WB menu node `Коляски`, so benchmark examples became real baby strollers. That can falsely suppress a good toy.

   Fix: `app/services/providers/wb_public.py` no longer shortens multiword subject names by first-word stem. `Коляски` no longer matches `Коляски для кукол`.

3. Missing subject-catalog fallback for renamed WB roots.

   Symptom: `Хозяйственные товары/Дозаторы для ванной` and `Товары для животных/Комбинезоны для животных` could not find menu paths and fell back to heuristic benchmark.

   Fix: added subject shard hints/root aliases for `Хозяйственные товары` and `Товары для животных`. These cases now use `wb_subject_catalog`.

4. Contradictory quantity output.

   Symptom: final verdict could be `RED`, while UI showed a large “recommended purchase” from retail fallback, e.g. 4646 units.

   Fix: raw yearly forecast remains in `forecast_units_year`, but action quantity now follows verdict: `RED = 0`, `YELLOW = capped test lot`, `GREEN = full forecast`.

## Remaining Weaknesses

1. Retail fallback is still broad.

   If no WB coefficient exists, `retail_fact` can come from a whole Raznotorg view/category rather than the exact model on photo. It is useful as context, but not a reliable model-level quantity.

   Proposal: show `forecast_source=retail_fact` as low-confidence and keep it as fallback only. For final quantity, require exact/stem retail match quality or an explicit category coefficient.

2. LEGO-compatible constructor benchmark is too strict.

   The MY-WORLD-compatible set is classified as `Конструкторы LEGO`, and benchmark top is dominated by strong LEGO/Minecraft cards. This can produce false RED for compatible/non-brand constructor sets.

   Proposal: split branded LEGO vs compatible constructors by brand/title tokens. If visual analogs are mostly non-brand/compatible, exclude official `LEGO` cards from benchmark or mark benchmark as brand-mismatch.

3. Doll stroller has no real WB catalog benchmark.

   After the false menu match fix, the system correctly avoids baby strollers but falls back to `visual_subset_in_subject`. This is safer, but still less reliable.

   Proposal: add a persistent subjectId→shard override or a paid/category API source for subjects not present in menu tree.

4. Visual search can be unstable on infographic-heavy photos.

   Dog jacket repeated runs moved from pet category to mixed/UNKNOWN. This is good fail-closed behavior, but field UX needs to tell buyer: “retake photo without text/collage, crop product”.

   Proposal: add a warning when `subject_vote_share < 0.7`, and show “photo too noisy” rather than only technical reasons.

5. Ozon mapping gaps remain visible.

   Several good categories still return `WB_ONLY`. This should cap confidence, but should not halve/penalize WB demand itself.

   Proposal: keep WB demand separate, and treat missing Ozon as market-coverage risk only.
